# Runtime Matrix

This matrix records the WSL-first and Jetson-Orin-targeted backend strategy. Status is intentionally conservative: a path is "implemented" only when this repository contains runnable scripts/client support. Runtime claims are limited to the exact smoke tests that have been observed.

## Initial Priority

1. llama.cpp GGUF on WSL with `llama-server`.
2. llama.cpp GGUF on Jetson through dusty-nv `llama_cpp` Docker containers or a Jetson-compatible build.
3. Optional notes for Ollama/NanoLLM after baseline logs exist.
4. No custom CUDA or TensorRT kernels in the first version.

## Model Notes

- Gemma 4 E2B-it: the verified WSL Q8 baseline is `configs/models/gemma4_e2b_q8.yaml` with pre-quantized `Q8_0` model and mmproj files from `ggml-org/gemma-4-E2B-it-GGUF`. The lower-memory Q4 path is `configs/models/gemma4_e2b_q4.yaml` with pre-built `Q4_K_M` artifacts from `mradermacher/gemma-4-E2B-it-GGUF`. Both Q8 and Q4 have passed WSL CUDA text, committed sample-image benchmark, and one-frame fake-stream checks with `CTX_SIZE=512`, `N_GPU_LAYERS=32`, `LLAMA_BATCH_SIZE=512`, `LLAMA_UBATCH_SIZE=512`, and one server slot. Gemma Q4 Jetson smoke passed with `CTX_SIZE=512`, `N_GPU_LAYERS=12`, batch/ubatch 512, `q8_0` KV cache, and the pinned Jetson llama.cpp image; the no-mmproj-offload comparison also passed but cut image throughput roughly in half. Local BF16-to-Q4 quantization exceeded this WSL memory budget, and Q8-to-Q4 re-quantization was stopped by user request because memory was insufficient. The earlier Gemma `LLAMA_UBATCH_SIZE=32` text-only setting triggered a llama.cpp non-causal attention assertion on image requests.
- MiniCPM-V 4.6: the WSL baseline is `configs/models/minicpmv46_q4.yaml` with official pre-built `Q4_K_M` model and F16 mmproj artifacts from `openbmb/MiniCPM-V-4.6-gguf`. WSL CUDA text, committed sample-image benchmark, and one-frame fake-stream checks passed with `CTX_SIZE=512`, `N_GPU_LAYERS=32`, `LLAMA_BATCH_SIZE=128`, `LLAMA_UBATCH_SIZE=32`, and one server slot. Jetson smoke passed with the same context/offload/batch shape plus `q8_0` KV cache and the pinned Jetson llama.cpp image. Local HF checkpoint conversion and local quantization are not baseline paths for this memory-constrained WSL host.
